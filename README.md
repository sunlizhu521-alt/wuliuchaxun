# 物流查询系统

静态 GitHub Pages 物流查询工具。页面在浏览器本地读取 Excel/CSV，业务表格默认保存到 IndexedDB，不上传到服务器。

## 页面

- `index.html`: 默认物流查询页，支持手动单条查询、地址表格拖拽导入、导入模板下载和结果导出。
- `dimension-library.html`: 维度表库，8 个固定槽位，保存到浏览器 IndexedDB。

## 维度表槽位

1. 发货地址
2. 商品信息表
3. 商品包装明细
4. 物流公司报价
5. 维度1
6. 维度2
7. 维度3
8. 维度4

## 查询规则

- 发货地址和顾客地址保持独立输入。
- 商品信息按物料编码必填查询，销售产品线、销售系列、型号由商品信息表自动带出，购买件数空值按 1 处理。
- 物料编码为空时不参与物流推荐；如果辅助条件匹配到多个物料，需要补充或修正商品信息。
- 商品包装明细按输入物料编码匹配；单件包裹数优先取该行 `件数`，总实际重量优先取该行 `总实际重量`，缺失时按包裹明细计算兜底，结果只展示该行最左侧物料编码。
- 商品包装明细支持一个商品多包裹，一行多组字段，默认模板提供包裹 1 到包裹 6。
- 物流公司报价从 `物流公司报价` 槽位读取多 sheet，sheet 名按 `发货地-物流公司名称` 解析；同一发货地只比较匹配 sheet。
- 总推荐物流计费重量 = 商品包装明细 `总体积` x 购买件数 / 当前物流 sheet 的 `泡比`，不与实际重量取大。
- 运费按当前物流 sheet 的 `首重（XXKG）` / `首重XXKG`、`续重（xx-xxKG）`、`续重（xxKG以上）` / `续重（xx以上）` 阶梯计算。
- 最终结果显示：销售产品线、销售系列、销售型号、物料编码（主）、商品名称、购买数量、单件包裹数、总实际重量、总推荐物流计费重量、推荐物流、预估费用、备选物流、备选物流费用。
- 推荐物流和备选物流按同一发货地匹配到的物流公司报价 sheet 计算费用后排序。
- GitHub Pages 默认不发布真实业务 Excel，`data/shared-library.json` 保持空记录。
